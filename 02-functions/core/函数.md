# Function 对象

## 属性

| 属性 | 说明 |
|------|------|
| `name` | 函数名；匿名函数取赋值变量或属性名 |
| `length` | 预期形参个数（不含剩余参数；默认参数之后不计数） |
| `caller` | 调用者（非标准，已废弃） |
| `prototype` | 仅普通函数有，用于 `new` 时的原型 |

## 原型方法

- `apply(thisArg, [args])`
- `call(thisArg, ...args)`
- `bind(thisArg, ...preset)`
- `toString()`

## 继承方法

`valueOf`、`hasOwnProperty`、`isPrototypeOf`、`propertyIsEnumerable` 等。

## 函数定义形式

- **函数声明**：存在提升（function hoisting）
- **函数表达式**：必须先定义后调用
- **箭头函数**：无 `arguments` / `prototype`，`this` 词法绑定
- **`new Function(...)`**：运行时由字符串创建，`[[Environment]]` 指向全局

```js
const fn = new Function('a', 'b', 'return a + b');
```

使用 `new Function` 的场景很少（动态模板、远程代码）。它无法闭包创建点的外部局部变量，但有助于避免意外引用与 minifier 问题。

## 参数

- **默认参数**：惰性求值；形成独立参数作用域
- **剩余参数** `...rest`：真数组，与 `arguments` 分离
- **解构默认值**：推荐 `function f({ x, y = 5 } = {})`

有默认参数时，命名参数与 `arguments` 不再同步。
