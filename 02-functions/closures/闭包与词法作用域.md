# 闭包与词法作用域

## 作用域

编程语言最基本的功能之一，就是能够存储变量中的值，并在之后访问或修改它。

**作用域**负责收集和维护由所有声明的标识符组成的查询集合，并实施一套严格的规则，确定当前执行代码对这些标识符的访问权限。

## JavaScript 的编译与执行

JavaScript 引擎并不在编译时将所有内容确定到机器码，而是采用**编译 + 解释**的混合模式。大致分为三个阶段：

1. **分词 / 词法分析**：将字符组成的字符串分解为有意义的代码块（词法单元 token）。
2. **解析 / 语法分析**：将词法单元流转换为由逐级嵌套元素组成的抽象语法树（AST）。
3. **代码生成**：将 AST 转换为可执行代码。

参与这一过程的主要角色：

| 角色 | 职责 |
|------|------|
| 引擎 | 负责 JavaScript 代码的编译与执行 |
| 编译器 | 负责词法分析、语法分析及代码生成 |
| 作用域 | 负责维护标识符的查询规则与访问权限 |

## 标识符查询：LHS 与 RHS

引擎执行代码时，会区分两种查询：

- **LHS（Left-Hand Side）**：赋值操作的左侧，查找变量的**容器**本身，以确认是否可以赋值。非严格模式下，若在作用域链中找不到，会在全局对象上创建该变量。
- **RHS（Right-Hand Side）**：读取变量的值，沿作用域链查找。找不到则抛出 `ReferenceError`。

两种常见错误：

- `ReferenceError`：作用域判别失败，标识符不存在。
- `TypeError`：作用域判别成功，但执行的操作非法或不合理（例如对 `null` 调用方法）。

## 词法作用域

**词法作用域**（Lexical Scope）也叫静态作用域：作用域在**代码书写时**就已确定，与函数在哪里被调用无关。

变量查找从当前位置出发，逐级向外层作用域查找，直到找到或到达最外层为止。这一过程称为**作用域嵌套**。

```js
const func = function () {
  if (true) {
    let age = 25;
  }
  console.log(age); // ReferenceError：age 只在 if 块内可见
};

func();
```

`let` / `const` 具有**块级作用域**，只在声明所在的 `{ }` 内有效。`var` 则只有函数作用域，没有块级作用域。

## 词法环境

在 JavaScript 中，每个运行的函数、代码块 `{ }` 或整个脚本，都关联一个称为**词法环境**的内部对象。

词法环境由两部分组成：

1. **环境记录（Environment Record）**：存储当前作用域内的局部变量、`let` / `const` 绑定，以及 `this` 等额外信息。
2. **外部词法环境的引用**：指向外层（当前花括号之外）的词法环境，形成作用域链。

因此：

- 「变量」本质上是环境记录这个内部对象的属性。
- 「访问或修改变量」意味着访问或修改词法环境的某个属性。
- 变量与当前执行的块 / 函数 / 脚本绑定；操作变量就是在操作对应环境记录上的属性。

## 闭包

### 定义

**闭包**是函数式编程语言的一个特性。通俗地说：

> 闭包是一个函数在创建时，允许自身访问并操作**声明时所在作用域**中的变量时所形成的作用域；即便该外层作用域在函数创建之后已经「离开」执行栈，闭包仍能访问那些变量。

函数会记住创建时的词法环境——在规范中通过内部的 `[[Environment]]` 引用实现——因此可以访问外部变量。

1. **JavaScript 中所有函数都是闭包**（只要函数引用了外部词法环境中的变量，或即便没有引用，函数本身也持有对外部环境的引用）；
2. 可选的技术细节：`[[Environment]]` 与词法环境链的查找机制。

> 例外：`new Function(...)` 创建的函数，其 `[[Environment]]` 指向全局环境而非当前词法环境，因此无法直接闭包外部局部变量。这是有意为之，能减少动态生成代码时的意外引用。

### 闭包的作用

- **延长外部变量的生命周期**：内部函数持有对外部变量的引用，阻止其被垃圾回收。
- **数据私有化**：通过闭包隐藏变量，只暴露受控的访问接口。
- **函数工厂与柯里化**：根据参数生成定制化的函数（见 [code/closure-factory.js](code/closure-factory.js)）。

```js
function sum(a) {
  return function (b) {
    return a + b;
  };
}

console.log(sum(2)(5)); // 7

function isBetween(a, b) {
  const max = Math.max(a, b);
  const min = Math.min(a, b);
  return function (value) {
    return value >= min && value <= max;
  };
}

function inArray(arr) {
  return function (value) {
    return arr.includes(value);
  };
}

const arr = [1, 2, 3, 4, 5, 6, 7];
console.log(arr.filter(isBetween(3, 6)));       // [3, 4, 5, 6]
console.log(arr.filter(inArray([1, 2, 10])));    // [1, 2]
```

### 经典陷阱：循环中的闭包

在 `var` 循环里直接创建函数，所有闭包往往共享同一个循环变量：

```js
// 错误示范：var 没有块级作用域，所有回调里的 i 最终都是 10
const shooters = [];
for (var i = 0; i < 10; i++) {
  shooters.push(function () {
    console.log(i);
  });
}
shooters[0](); // 10
```

常见修复方式：

```js
// 方式一：用 let，每次迭代有独立的词法绑定
for (let i = 0; i < 10; i++) {
  shooters.push(function () {
    console.log(i);
  });
}

// 方式二：IIFE 捕获当前值
for (var i = 0; i < 10; i++) {
  shooters.push((function (j) {
    return function () {
      console.log(j);
    };
  })(i));
}

// 方式三：在循环体内用 let 复制当前值（见 code/loop-closure.js 中 makeArmyCopy）
let i = 0;
while (i < 10) {
  const j = i;
  shooters.push(function () {
    console.log(j);
  });
  i++;
}
```

## 概念关系小结

```
编译（分词 → 解析 → 代码生成）
        ↓
词法作用域（书写时确定的作用域规则）
        ↓
词法环境（环境记录 + 外部引用，形成作用域链）
        ↓
闭包（函数通过 [[Environment]] 记住创建时的词法环境）
```
