# 第6章 对象

## 创建对象

```js
// 对象字面量
let obj = { x: 1, y: 2 };

// new 构造函数
let obj2 = new Object();
let date = new Date();

// Object.create()
let obj3 = Object.create(proto);        // 指定原型
let obj4 = Object.create(null);         // 无原型对象

// Object.fromEntries() (ES2019)
let obj5 = Object.fromEntries([['a', 1], ['b', 2]]);
```

## 属性操作

```js
obj.x           // 点访问
obj['x']        // 方括号访问（支持变量和表达式）
obj.x = 1       // 设置属性
delete obj.x    // 删除属性
'x' in obj      // 检查属性是否存在（含继承）
obj.hasOwnProperty('x')  // 自身属性检查
```

## 属性描述符

```js
Object.getOwnPropertyDescriptor(obj, 'x');
Object.defineProperty(obj, 'x', {
  value: 1,
  writable: true,       // 可写
  enumerable: true,     // 可枚举
  configurable: true    // 可配置（可删除、可修改特性）
});
Object.defineProperties(obj, { /* 多个属性 */ });
```

## 访问器属性 (getter/setter)

```js
let obj = {
  _x: 0,
  get x() { return this._x; },
  set x(val) { this._x = val; }
};
```

## 属性枚举

```js
for (let key in obj)        // 可枚举属性（含继承），不含 Symbol
Object.keys(obj)            // 自身可枚举属性名（不含 Symbol）
Object.values(obj)          // 自身可枚举属性值
Object.entries(obj)         // 自身可枚举键值对
Object.getOwnPropertyNames(obj)    // 自身所有属性名（不含 Symbol）
Object.getOwnPropertySymbols(obj)  // 自身 Symbol 属性
Reflect.ownKeys(obj)               // 自身所有属性（含 Symbol）
```

## 扩展与合并

```js
Object.assign(target, ...sources);  // 浅复制可枚举属性
let merged = { ...a, ...b };        // 展开运算符
```

## 对象方法

```js
obj.toString()         // 默认返回 "[object Object]"
obj.toLocaleString()
obj.valueOf()          // 转为原始值
obj.toJSON()           // 自定义 JSON 序列化
```

## 计算属性名 (ES6)

```js
let prop = 'name';
let obj = { [prop]: 'value' };  // { name: 'value' }
```

## 简写语法 (ES6)

```js
let x = 1, y = 2;
let obj = { x, y };           // { x: 1, y: 2 }
let methods = {
  foo() {},                   // foo: function() {}
  *gen() {}                   // generator 方法
};
```

## Symbol 属性

Symbol 属性不会被常规枚举发现，但可以通过上述专门的 Symbol API 访问。用作非字符串属性名，避免命名冲突。

## 对象不可变性控制

```js
Object.preventExtensions(obj);  // 禁止添加新属性
Object.seal(obj);               // 禁止添加/删除属性，标记 configurable: false
Object.freeze(obj);             // 禁止添加/删除/修改属性值（浅冻结）
```
- 以上均为**浅操作**，嵌套对象不受影响
- `Object.isExtensible(obj)`、`Object.isSealed(obj)`、`Object.isFrozen(obj)` 检查状态
- 被冻结的对象严格模式下修改会抛 TypeError，非严格模式下静默失败

## 容易弄错的点

1. **`Object.assign` 是浅拷贝**：嵌套对象共享引用
   ```js
   let a = { x: { y: 1 } };
   let b = Object.assign({}, a);
   b.x.y = 2;  // a.x.y 也变成了 2！
   ```

2. **`in` 检查继承属性**：`"toString" in {}` → `true`。若只检查自身属性用 `hasOwnProperty()`

3. **`for/in` 枚举继承的可枚举属性**：这与 `Object.keys` 不同，后者只返回自身的

4. **`delete` 不释放内存**：只删除属性引用，不影响原型链上的同名属性

5. **对象的键总是字符串（或 Symbol）**：`obj[1]` 实际上访问的是 `obj["1"]`。Map 才支持任意类型键

6. **getter/setter 和同名数据属性不能共存**：一个属性要么是数据属性(value+writable)，要么是访问器属性(get+set)，不能同时

7. **`Object.create(null)` 创建的对象没有 `hasOwnProperty`**：需用 `Object.hasOwn(obj, prop)` (ES2022) 或 `Object.prototype.hasOwnProperty.call(obj, prop)`

8. **`{ __proto__: ... }` 是唯一会改变原型的对象字面量语法**：其他属性名不会。且同时使用多个 `__proto__` 键是语法错误
