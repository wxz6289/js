# 第10章 模块

## 模块 vs 常规脚本

| 特性 | 模块 | 常规脚本 |
|------|------|---------|
| 顶级变量作用域 | 模块内（不污染全局） | 全局 |
| 严格模式 | 自动 | 需手动 `"use strict"` |
| 导出/导入 | `export`/`import` | 不支持 |
| this（顶级） | `undefined` | 全局对象 |
| 加载方式 | 默认 defer | 同步阻塞 |
| 跨源限制 | 是（需 CORS） | 否 |
| file: 协议 | 不支持 | 支持 |

## 导出

```js
// 命名导出（可多个）
export const name = 'value';
export function fn() {}
export { a, b, c };
export { a as alias };

// 默认导出（每个模块最多一个）
export default function() {}
export default class {}
export { fn as default };
```

- 导出的是**绑定引用**（live binding），不是值的快照
- `export` 只能出现在模块顶级，不能在函数/块中

## 导入

```js
// 命名导入
import { a, b } from './module.js';
import { a as alias } from './module.js';

// 默认导入
import fn from './module.js';

// 命名空间导入
import * as mod from './module.js';

// 仅执行副作用
import './init.js';

// 混合
import fn, { a, b } from './module.js';
```

- `import` 提升到模块顶部
- 导入路径必须是字符串字面量
- 裸模块说明符（bare specifier）需要 import map 或打包工具解析

## 重导出

```js
export { a, b } from './other.js';
export { default as Other } from './other.js';
export * from './other.js';        // 重导出所有命名导出（不含默认）
export * as ns from './other.js';  // ES2020
```

## 动态导入

```js
const module = await import('./module.js');
const { a, b } = await import(`./lang-${lang}.js`);
```
- 返回 Promise
- 可以在函数/条件/循环中使用
- 支持表达式路径
- 适合代码分割（code splitting）

## 在 HTML 中使用

```html
<script type="module" src="app.js"></script>
<script type="module">
  import { fn } from './utils.js';
</script>
<script nomodule src="legacy.js"></script>
```

- `type="module"` 的脚本默认 defer
- 可加 `async` 使模块在下载完毕后立即执行
- `nomodule` 在不支持模块的旧浏览器中执行

## ES Module vs CommonJS

| 特性 | ES Module | CommonJS |
|------|-----------|----------|
| 环境 | 浏览器+Node | Node（默认） |
| 语法 | `export`/`import` | `require`/`module.exports` |
| 加载时机 | 静态分析，编译时确定 | 运行时加载 |
| 绑定类型 | live binding（值的引用） | 值的拷贝 |
| 顶层this | `undefined` | `exports` 对象 |
| 循环依赖 | 自动处理 | 用户需手动处理 |

## 容易弄错的点

1. **ESM 导出的是 live binding（值的引用），不是值的快照**：导入方会看到导出方对变量的实时修改。CommonJS 导出的是值的拷贝
   ```js
   // counter.js (ESM)
   export let count = 0;
   export function increment() { count++; }
   // main.js
   import { count, increment } from './counter.js';
   increment(); console.log(count); // 1（实时反映变化！）
   ```

2. **ESM 的 `import` 提升到模块顶部**：即使在 `if` 或 `try` 块中写 `import`，也会在模块顶部执行。需要条件加载请用 `import()`

3. **模块脚本受 CORS 限制**：`<script type="module">` 从 `file://` 协议打开会因跨域报错，需通过 HTTP 服务器访问

4. **在浏览器中，模块路径必须写完整扩展名 `.js`**：`import './foo'` 不会自动补全 `.js`（Node.js 的 ESM 也需要完整扩展名）

5. **CommonJS 中 `require` 的结果会被缓存**：多次 `require` 同一模块返回同一个对象。若需每次获取新实例，导出工厂函数

6. **动态 `import()` 返回的是模块的命名空间对象**：`const m = await import('...'); m.default` 才是默认导出，不是 `m` 本身

7. **`export default` 和 `import` 不匹配**：
   ```js
   // 导出方 export default 123
   import val from './m.js';   // val = 123 ✅
   import { default as val } from './m.js'; // 等价
   import * as m from './m.js';  // m.default = 123（不是 m = 123）
   ```

8. **在 Node 中混用 CJS 和 ESM**：ESM 可以通过 `import` 导入 CJS 模块，但 CJS 不能用 `require` 导入 ESM（需用 `import()`）
