# 第14章 元编程

## Symbol

Well-known Symbols 用于自定义语言行为：

| Symbol | 用途 |
|--------|------|
| `Symbol.iterator` | 定义默认迭代器 |
| `Symbol.asyncIterator` | 定义异步迭代器 |
| `Symbol.hasInstance` | 自定义 instanceof 行为 |
| `Symbol.toPrimitive` | 定义对象→原始值转换 |
| `Symbol.toStringTag` | 自定义 Object.prototype.toString 标签 |
| `Symbol.species` | 控制派生对象的构造函数 |
| `Symbol.isConcatSpreadable` | 控制 Array.concat 是否展开 |
| `Symbol.unscopables` | 排除 with 语句中的属性 |
| `Symbol.match/replace/search/split` | 自定义字符串匹配行为 |

## Proxy (ES6)

```js
const proxy = new Proxy(target, {
  get(target, prop, receiver) { /* 拦截属性读取 */ },
  set(target, prop, value, receiver) { /* 拦截属性设置 */ },
  has(target, prop) { /* 拦截 'in' 操作符 */ },
  deleteProperty(target, prop) { /* 拦截 delete */ },
  apply(target, thisArg, args) { /* 拦截函数调用 */ },
  construct(target, args, newTarget) { /* 拦截 new */ },
  getPrototypeOf(target) { /* 拦截 Object.getPrototypeOf */ },
  setPrototypeOf(target, proto) { /* 拦截 Object.setPrototypeOf */ },
  defineProperty(target, prop, desc) { /* 拦截 Object.defineProperty */ },
  getOwnPropertyDescriptor(target, prop) { /* 拦截 getOwnPropertyDescriptor */ },
  ownKeys(target) { /* 拦截 Reflect.ownKeys */ },
  preventExtensions(target) { /* 拦截 Object.preventExtensions */ },
  isExtensible(target) { /* 拦截 Object.isExtensible */ }
});
```

### 不可配置的代理陷阱
- `getPrototypeOf` 必须是 `getPrototypeOf` 或 `getPrototypeOf` 的返回值
- `setPrototypeOf` 在不可扩展目标上返回 false

## Reflect API (ES6)

`Reflect` 提供与 Proxy 陷阱一一对应的方法，使默认行为可以在代理中方便调用：

```js
Reflect.get(obj, prop)
Reflect.set(obj, prop, value)
Reflect.has(obj, prop)
Reflect.deleteProperty(obj, prop)
Reflect.construct(Constructor, args)
Reflect.apply(fn, thisArg, args)
Reflect.ownKeys(obj)
Reflect.defineProperty(obj, prop, desc)
// ...
```

## new.target

在函数/构造函数中检测是否通过 `new` 调用：

```js
function Foo() {
  if (!new.target) throw new Error('must use new');
  console.log(new.target === Foo); // true
}
```

## 容易弄错的点

1. **Proxy 陷阱必须遵守不变量(invariants)**，否则抛 TypeError：
   - 不可配置+不可写的属性，`set` 陷阱不能修改其值
   - 不可扩展对象，`ownKeys` 不能返回额外属性
   - 不可配置属性，`deleteProperty` 返回 false 时报错

2. **Reflect 方法与对应的 Proxy 陷阱返回值不同**：`Reflect.set` 返回布尔值（表示是否成功），而直接赋值不返回。写 Proxy 陷阱时用 `return Reflect.set(...)` 可传达成败

3. **`Symbol.for` vs `Symbol`**：`Symbol.for("x")` 全局共享且可被 `Symbol.keyFor()` 获取 key；`Symbol("x")` 每次创建新符号，`Symbol.keyFor()` 返回 undefined

4. **Proxy 拦截 `in` 操作符用 `has` 陷阱**，而非 `get` 陷阱；拦截 `for/in` 需要 `ownKeys` + `getOwnPropertyDescriptor`

5. **`ownKeys` 陷阱在 `Object.keys()`、`for/in`、`Object.getOwnPropertyNames()`、`Object.getOwnPropertySymbols()`、`Reflect.ownKeys()` 时都会触发**

6. **Proxy 包装后 `===` 比较的是引用**：`proxy !== target`。要让其他代码"看不到"代理，需要非常小心

7. **`Symbol.toPrimitive` 优先级最高**：定义了它，`valueOf()` 和 `toString()` 在隐式类型转换时就不会被调用了（除非显式调用）
