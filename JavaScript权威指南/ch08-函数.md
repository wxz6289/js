# 第8章 函数

## 函数定义方式

```js
// 函数声明（提升）
function add(a, b) { return a + b; }

// 函数表达式
const add = function(a, b) { return a + b; };

// 箭头函数 (ES6)
const add = (a, b) => a + b;

// 构造函数（不推荐）
const add = new Function('a', 'b', 'return a + b');
```

## 函数调用方式

1. **作为函数调用**：`fn()` — this 指向全局对象（严格模式为 undefined）
2. **作为方法调用**：`obj.fn()` — this 指向调用对象
3. **作为构造函数调用**：`new Fn()` — this 指向新创建的对象
4. **call/apply 调用**：`fn.call(ctx, a, b)` / `fn.apply(ctx, [a, b])`
5. **bind 绑定**：`fn.bind(ctx)` — 返回绑定了 this 的新函数

## 参数

```js
// 默认参数 (ES6)
function f(a = 1, b = a * 2) {}

// 剩余参数 (ES6)
function f(...args) {}    // args 是真数组，不是类数组

// arguments（类数组对象，严格模式下为静态副本）
```

## 箭头函数特殊性

- 没有自己的 `this`（继承外层作用域）
- 没有 `arguments` 对象
- 不能用作构造函数（不能 new）
- 没有 `prototype` 属性
- 不能用作 generator

## 闭包

函数捕获其定义时所在作用域中的变量，形成闭包。内部函数可以访问外部函数的变量，即使外部函数已返回。

## 函数属性与方法

```js
fn.name          // 函数名
fn.length        // 期望参数个数（不含默认参数和剩余参数）
fn.toString()    // 返回函数源码字符串
fn.call(ctx, ...args)
fn.apply(ctx, argsArray)
fn.bind(ctx, ...args)
```

## 高阶函数

函数可以接受函数作为参数，也可以返回函数：

```js
function timed(f) {
  return function(...args) {
    console.time(f.name);
    try { return f(...args); }
    finally { console.timeEnd(f.name); }
  };
}
```

## 记忆化 (Memoization)

缓存函数计算结果，避免重复计算：

```js
function memoize(fn) {
  const cache = new Map();
  return function(arg) {
    if (cache.has(arg)) return cache.get(arg);
    const result = fn.call(this, arg);
    cache.set(arg, result);
    return result;
  };
}
```

## IIFE（立即调用函数表达式）

```js
(function() { /* ... */ })();
(() => { /* ... */ })();
```
ES6 块作用域（let/const）使 IIFE 的必要性大大降低。

## 尾调用优化 (TCO, ES6)

严格模式下，如果函数的最后一步是调用另一个函数（且不使用其返回值做额外计算），引擎可以不增加调用栈帧：

```js
"use strict";
// 尾调用（可优化）
function tail(x) { return next(x); }

// 非尾调用（不能优化—返回值参与了计算）
function notTail(x) { return next(x) + 1; }
```
- 只有 Safari/JSC 完整实现了 TCO，V8 和 SpiderMonkey 未实现
- 不要依赖此特性做深层递归优化，如果需要用迭代或 trampoline

## 容易弄错的点

1. **this 取决于调用方式，不是定义位置**（箭头函数例外）——这是 JS 最核心的易错点
   ```js
   const obj = {
     name: 'obj',
     fn() { console.log(this.name); }
   };
   const f = obj.fn;
   f();     // undefined（严格模式）或抛出——this 丢失了！
   obj.fn(); // 'obj'
   ```

2. **闭包中的循环陷阱**：`var` 没有块作用域，异步回调引用的是循环结束后的值
   ```js
   for (var i = 0; i < 3; i++) {
     setTimeout(() => console.log(i), 0); // 3, 3, 3!
   }
   // 解决方法：把 var 改为 let；或用 IIFE 创建独立作用域
   ```

3. **箭头函数不能用作方法**（会丢失 this）和不能用作构造函数（`new` 会抛 TypeError）

4. **`arguments` 是类数组，不是真数组**：不能用 `map`、`forEach` 等。用剩余参数 `...args` 替代（且是真数组）

5. **默认参数有自己的作用域**：`function f(x, y = x)` 中 y 的默认值可以引用前面的 x。但 `let x = 1; function f(y = x) {}` 中的 x 是外部变量

6. **`fn.bind()` 返回的新函数不能被再次 bind**：`fn.bind(a).bind(b)` 的 this 仍是 a（第一次绑定的值）

7. **构造函数忘记 `new` 不会报错**（非严格模式）：`let obj = Person();` 中 this 指向全局，返回 undefined。ES6 class 修复了此问题（不用 new 调用会抛 TypeError）

8. **函数声明提升但函数表达式不提升**：
   ```js
   foo(); // OK
   bar(); // TypeError: bar is not a function
   function foo() {}
   var bar = function() {};
   ```
