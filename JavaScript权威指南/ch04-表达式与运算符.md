# 第4章 表达式与运算符

## 表达式

- **主表达式**：字面量、变量引用、`this`
- **对象/数组初始化表达式**：`{}`, `[]`
- **函数定义表达式**：`function() {}`, `() => {}`
- **属性访问**：`.identifier`, `[expr]`
- **条件表达式**：`expr ? a : b`
- **调用表达式**：`fn(args)`
- **对象创建表达式**：`new Class()`

## 运算符分类

### 算术运算符
`+`, `-`, `*`, `/`, `%`, `**` (指数，ES2016)

### 一元运算符
`+x`, `-x`, `++x`, `x++`, `--x`, `x--`

### 位运算符
`&`, `|`, `^`, `~`, `<<`, `>>`, `>>>`

### 比较运算符
`<`, `>`, `<=`, `>=`, `==`, `!=`, `===`, `!==`

### 逻辑运算符
`&&` (短路与), `||` (短路或), `!` (逻辑非)

### 空值合并 (ES2020)
`??` — 左侧为 `null` 或 `undefined` 时返回右侧

### 赋值运算符
`=`, `+=`, `-=`, `*=`, `/=`, `%=`, `**=`, `&&=`, `||=`, `??=`

### 其他
- `in`：属性是否存在
- `instanceof`：是否属于某类
- `typeof`：返回操作数类型字符串
- `delete`：删除属性
- `void`：求值并返回 `undefined`
- `,`：逗号运算符，返回右侧值
- `?.`：可选链 (ES2020)
- `...`：展开/剩余运算符

## 求值顺序

- 运算符优先级决定结合方式
- 结合性决定同优先级运算符的执行方向
- 子表达式求值顺序通常从左到右

## 容易弄错的点

1. **`+` 运算符的双重语义**：有字符串操作数→字符串连接；否则→数值加法。这是最常见 bug 来源
   ```js
   1 + 2 + "3"   // "33"  (不是 "123")
   "1" + 2 + 3   // "123" (不是 "15")
   1 + {}        // "1[object Object]"
   true + true   // 2（不是 "truetrue"）
   ```

2. **`==` 比较对象时比较的是引用**：`[] == []` → `false`，`{} == {}` → `false`

3. **`??` 不能与 `&&` 或 `||` 直接混用**：`a && b ?? c` → SyntaxError。必须加括号：`(a && b) ?? c`

4. **`||` vs `??` 的关键区别**：`0 || "default"` → `"default"`（0 是 falsy），`0 ?? "default"` → `0`（0 不是 null/undefined）

5. **`delete` 的返回值陷阱**：`delete` 操作成功返回 `true`，即使属性不存在也返回 `true`（严格模式下删除不可配置属性才抛错）

6. **`>>` vs `>>>`**：`>>` 带符号右移（保留负号），`>>>` 无符号右移（左侧补0，负数变大正数）

7. **后置 `++` 返回旧值**：`let x = 1; let y = x++;`  → `y === 1, x === 2`

8. **`1 < 2 < 3` 返回 `true`**，但 `3 > 2 > 1` 返回 `false`！因为 `3 > 2` → `true`，然后 `true > 1` → `1 > 1` → `false`
