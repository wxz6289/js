# 第5章 语句

**语句**驱动程序执行：控制流、声明绑定、异常处理。语句以分号或 `}` 结束；与**表达式**不同，语句本身通常不能嵌套为子表达式（表达式语句是例外）。

---

## 语句分类

| 类别 | 语句 | 说明 |
|------|------|------|
| 表达式语句 | `x = 1;`、`foo();`、`++i;` | 求值后丢弃结果 |
| 块语句 | `{ ... }` | 组合多条语句，形成块级作用域 |
| 空语句 | `;` | 占位，无操作 |
| 声明 | `var` / `let` / `const`、`function`、`class` | 创建绑定 |
| 条件 | `if`、`switch` | 分支选择 |
| 循环 | `while`、`do...while`、`for`、`for...in`、`for...of` | 重复执行 |
| 跳转 | `break`、`continue`、`return`、`throw` | 改变控制流 |
| 标签 | `label: statement` | 供 `break` / `continue` 指定目标 |
| 异常 | `try` / `catch` / `finally` | 错误处理 |
| 指令 | `"use strict"` | 启用严格模式 |
| 其他 | `debugger`、`with`（已废弃） | 调试 / 历史语法 |

---

## 表达式语句与块语句

### 表达式语句

赋值、函数调用、自增等表达式后加分号即构成语句，用于**副作用**（修改变量或环境状态）：

```js
(x + 1);
foo();
++i;
a = b = 1;
```

### 块语句

大括号 `{ }` 将多条语句组合为一条复合语句；块内语句通常以分号结尾，**块本身不以分号结尾**。

```js
{
  let x = 1;
  console.log(x);
}
```

`if`、`for`、`while` 等后的 `{ }` 语法上可省略，但**强烈建议始终使用**，避免悬挂 `else` 等问题。

### 空语句

单独一个分号 `;` 是空语句，常见于 `for` 循环体占位：

```js
for (let i = 0; i < n; i++);
```

## 条件语句

### if / else if / else

```js
if (condition) {
  // ...
} else if (other) {
  // ...
} else {
  // ...
}
```

`condition` 必须是表达式；`if (let x = 1)` 非法，但 `if ((x = 1))` 合法。

**悬挂 else**：无花括号时，`else` 与**最近的**未配对 `if` 绑定：

```js
if (a)
  if (b)
    console.log('a && b');
  else
    console.log('绑到内层 if');  // 不是外层 if 的 else
```

### switch

```js
switch (expression) {
  case value1:
    // ...
    break;
  case value2:
    // ...
    break;
  default:
    // ...
}
```

| 要点 | 说明 |
|------|------|
| 匹配方式 | `switch` 表达式与 `case` 使用 **`===` 全等**比较 |
| `case` 表达式 | 通常为常量表达式 |
| 穿透（fall-through） | 无 `break` 时继续执行下一 `case` |
| `default` 位置 | 可在任意位置；仅当所有 `case` 都不匹配时执行 |
| 作用域 | 各 `case` 共享外层块；`let`/`const` 重复声明会报错，可用 `{ }` 包裹子句 |

```js
switch (1) {
  case 1: console.log(1);
  case 2: console.log(2);  // 无 break，也会执行
}
```

## 循环语句

### while 与 do...while

```js
while (condition) { /* ... */ }

do {
  /* 至少执行一次 */
} while (condition);  // 末尾分号不能省
```

### for

```js
for (initialization; condition; afterthought) {
  // ...
}
```

| 部分 | 执行次数 |
|------|----------|
| 初始化 | 1 次 |
| 条件测试 | 每次迭代前 |
| 更新表达式 | 每次迭代后 |

三部分均可省略；`for (;;)` 为无限循环。头部 `let`/`const` 在**每次迭代**创建新的块级绑定（与经典 `for` 中 `var` 不同）。

### for...in

遍历对象的**可枚举属性名**（字符串键）：

```js
for (const key in obj) { /* ... */ }
```

| 规则 | 说明 |
|------|------|
| 右侧表达式 | 非对象则先转为对象；`null`/`undefined` 会抛错 |
| 不枚举 | Symbol 属性、不可枚举属性 |
| 会枚举 | 继承链上的可枚举属性 |
| 动态修改 | 循环中删除未访问属性则不再枚举；新增属性可能被枚举 |
| 遍历顺序 | ES2020+ 规定整数键升序、其余字符串键按插入顺序，但跨引擎仍不宜依赖 |

**不要**用 `for...in` 遍历数组（索引为字符串、可能含继承属性）；优先 `for...of` 或数组方法。过滤继承属性用 `Object.hasOwn(obj, key)`。

### for...of

遍历**可迭代对象**的元素值；`of` 后的表达式只求值一次：

```js
for (const val of arr) { /* ... */ }
for (const [k, v] of map) { /* ... */ }
```

普通对象默认不可迭代，`for (let v of obj)` → `TypeError`。可遍历数组、字符串、Map、Set、TypedArray、arguments、DOM 集合等。

### for await...of

在 `async` 函数中遍历**异步可迭代对象**：

```js
async function read(stream) {
  for await (const chunk of stream) {
    process(chunk);
  }
}
```

## 跳转语句

### break

退出最近的 `switch` 或循环；可配合标签跳出外层循环。

### continue

跳过当前迭代剩余部分，进入下一次迭代。

`break` / `continue` 与标签之间**不能换行**（否则 ASI 插入分号，标签失效）。二者均**不能**跨函数边界，不能替代 `return`。

### return

结束当前函数执行并返回（可选）表达式值。

### throw

抛出异常，由最近的 `catch` 处理或沿调用栈传播。

**ASI 陷阱**：`return`、`throw`、`break`、`continue` 后直接换行会插入分号：

```js
return
  { x: 1 };   // 实际 return; → undefined

return { x: 1 };  // 正确
```

## 标签语句

语法：`label: statement`

为语句（通常是循环）命名，供 `break label` / `continue label` 使用：

```js
outer: for (let i = 0; i < 3; i++) {
  inner: for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) break outer;
  }
}
```

| 要点 | 说明 |
|------|------|
| 作用域 | 标签仅在定义它的语句块/函数内有效 |
| 嵌套 | 嵌套语句不能使用相同标签；非嵌套语句可以 |
| 多标签 | 一条语句可有多个标签：`a: b: statement` |
| 限制 | 不能用于 `let`/`const` 声明前；**不是** `goto` |
| 现代写法 | 多层循环优先提取函数 + `return` |

## try / catch / finally

```js
try {
  // 可能抛错的代码
} catch (err) {
  // 处理 err
} finally {
  // 无论是否抛错都会执行
}
```

| 要点 | 说明 |
|------|------|
| `catch` 绑定 | `catch (e)` 中 `e` 为块级绑定；ES2019+ 可省略绑定 `catch { }` |
| `finally` | 常用于释放资源；其中 `return` 会覆盖 `try`/`catch` 的返回值 |
| 可选组合 | `try/finally`（无 `catch`）、`try/catch`（无 `finally`）均合法 |

```js
try {
  return 1;
} finally {
  return 2;  // 最终返回 2
}
```

## 声明语句（概要）

| 语句 | 作用域 | 提升 |
|------|--------|------|
| `var` | 函数 / 全局 | 提升并初始化为 `undefined` |
| `let` / `const` | 块级 | 提升但处于 TDZ |
| `function` | 函数 / 块 | 函数声明整体提升 |
| `class` | 块级 | 提升但处于 TDZ；**不会**像函数声明那样提前可用 |

## 其他语句

### debugger

```js
debugger;  // 开发者工具打开时在此暂停
```

生产代码应移除。

### with（已废弃）

```js
with (obj) {
  // 块内属性查找先查 obj
}
```

严格模式禁止；阻碍优化。用解构或临时变量替代。

## 严格模式 `"use strict"`

**指令**与常规语句的区别：

- 由特定字符串字面量构成的表达式语句
- 不含语言关键字
- 只能出现在脚本或函数体**最开头**（其他语句之前）

### 严格代码的判定

- 函数体含 `"use strict"`，或在严格代码中定义
- 传给 `eval()` 的字符串含 `"use strict"`，或在严格代码中调用 `eval()`
- **所有 class 体与 ES 模块代码默认为严格模式**

严格模式修复语言缺陷、加强错误检查；非默认，以便遗留代码继续运行。

### 与非严格模式的主要区别

| 方面 | 严格模式 |
|------|----------|
| `with` | 禁止 |
| 变量 | 必须显式声明（赋值未声明变量抛 `ReferenceError`） |
| `this` | 独立函数调用时为 `undefined`（`call`/`apply` 传入 `null`/`undefined` 不再替换为全局对象） |
| 属性写入 | 不可写属性、不可扩展对象禁止新增属性 |
| `eval` | 在独立作用域中创建绑定，不泄漏到调用者 |
| `arguments` | 与形参解耦，为传入值的静态副本 |
| `delete` | 未限定标识符 → `SyntaxError`；不可配置属性 → `TypeError` |
| 字面量 | 对象同名属性、函数同名参数 → 语法错误 |
| 八进制 | 禁止 `0123`（仅允许 `0o` 前缀） |
| 保留 | `eval`、`arguments` 不可作标识符或重新赋值 |
| 调用栈 | `arguments.callee`、`arguments.caller`、函数 `.caller` / `.arguments` 抛 `TypeError` |

## 容易弄错的点

1. **`switch` 使用 `===` 全等比较**，不是 `==`。`case` 匹配后继续穿透，除非遇到 `break`

   ```js
   switch (1) {
     case 1: console.log(1);
     case 2: console.log(2);  // 也会输出
   }
   ```

2. **`for/in` 枚举继承属性**：用 `Object.hasOwn(obj, key)` 过滤，或改用 `Object.keys()` / `for...of`

3. **标签只能用于 `break`/`continue`**，不能 `goto`；作用于带标签的循环或语句块

4. **`do/while` 末尾必须有分号**：`do {} while (cond);`

5. **`for/of` 不能用于普通对象**：`for (let v of obj)` → `TypeError`

6. **`for/in` 遍历数组**：索引为字符串，且可能包含继承属性；遍历数组用 `for...of`

7. **严格模式下独立调用的函数 `this` 是 `undefined`**

   ```js
   function fn() { 'use strict'; console.log(this); }
   fn(); // undefined（非严格模式为 global/window）
   ```

8. **`eval` 在严格模式下有独立作用域**：非严格模式下 `eval` 内的 `var`/函数声明会泄漏到调用者作用域

9. **`return` 后换行触发 ASI**：`return` 与返回值必须在同一行，或写成 `return ({ x: 1 })`

10. **循环 + `var` + 闭包**：所有闭包共享同一变量；用 `let` 或 IIFE

    ```js
    const funcs = [];
    for (let i = 0; i < 3; i++) {
      funcs.push(() => i);
    }
    funcs[1](); // 1（var 则均为 3）
    ```

11. **`finally` 中的 `return` 覆盖 `try`/`catch` 的返回值**——资源清理时注意不要在 `finally` 中随意 `return`

12. **class 与 ES module 代码永远是严格模式**，无需也不应再写 `"use strict"`
