# 第2章 词法结构

## 字符集

- JavaScript 使用 Unicode 字符集（UTF-16 编码）
- 大小写敏感
- 标识符支持 Unicode 字母、数字、`$`、`_`

## 空白与换行

- 空格、制表符、换行符通常被忽略
- 换行符可作为语句终止符（ASI 自动分号插入机制）

## 注释

```js
// 单行注释
/* 多行注释 */
```

## 字面量

- 数值字面量：`42`, `3.14`, `0xff`, `1e6`
- 字符串字面量：`"hello"`, `'world'`, `` `template` ``
- 正则表达式字面量：`/pattern/`
- 模板字面量：支持多行和插值 `${expr}`

## 标识符与保留字

- 首字符必须是字母、`_` 或 `$`
- 后续字符可以是字母、数字、`_`、`$`
- 不能使用保留字（`class`、`const`、`function` 等）
- ES5+ 保留字在特定上下文中被限制

## 可选分号 (ASI)

- JavaScript 会自动插入分号
- 三条规则：换行前 token 无法解析时插入、`}` 前插入、文件末尾插入
- **陷阱**：`return`、`throw`、`break`、`continue`、`++`、`--` 后换行会触发 ASI

## 容易弄错的点

1. **ASI 经典坑**：`return` 后换行会自动插入分号，导致返回 `undefined`
   ```js
   // 错误！返回 undefined
   return
     { x: 1 };   // ASI 插入: return; { x: 1 };
   // 正确写法
   return {
     x: 1
   };
   ```

2. **标识符中的零宽字符**：Unicode 允许零宽连接符和零宽非连接符在标识符中，但肉眼不可见，可能导致神秘错误

3. **Unicode 转义**：`\uXXXX` 转义序列在字符串和标识符中均有效，但 `\u{XXXXX}`（码点转义）仅支持 ES6+

4. **注释嵌套陷阱**：`/*` 不能嵌套，遇到第一个 `*/` 就会结束
   ```js
   /* 外层 /* 内层 */  这里已经不在注释里了 */  // 语法错误
   ```

5. **正则字面量歧义**：以 `/` 开头的行可能被解析为正则或除法运算符，ASI 在这里表现特殊——正则不会被自动插入分号打断
