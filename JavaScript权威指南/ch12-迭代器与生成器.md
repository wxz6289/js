# 第12章 迭代器与生成器

## 迭代协议

### 可迭代协议
对象实现 `[Symbol.iterator]()` 方法，返回一个迭代器对象。

### 迭代器协议
迭代器对象有 `next()` 方法，返回 `{ value, done }`。

```js
let range = {
  from: 1, to: 5,
  [Symbol.iterator]() {
    return {
      current: this.from,
      last: this.to,
      next() {
        if (this.current <= this.last) {
          return { value: this.current++, done: false };
        } else {
          return { done: true };
        }
      }
    };
  }
};
for (let n of range) console.log(n); // 1 2 3 4 5
```

## 内置可迭代对象

- Array、String、Map、Set、TypedArray
- NodeList、arguments
- `for/of` 循环、展开运算符 `...`、解构赋值都依赖迭代协议

## 生成器 (Generator)

```js
function* gen() {
  yield 1;
  yield 2;
  yield 3;
}
let g = gen();
g.next(); // { value: 1, done: false }
g.next(); // { value: 2, done: false }
g.next(); // { value: 3, done: false }
g.next(); // { value: undefined, done: true }
```

- `yield` 暂停执行，返回 `{ value, done }`
- `yield* iterable` 委托给另一个迭代器/生成器
- `g.next(arg)` 向生成器传入值（替换上一次 yield 表达式）
- `g.return(value)` 提前终止
- `g.throw(err)` 向生成器内抛出异常

## 生成器的高级用法

```js
// 生成器作为可迭代对象
for (let v of gen()) {}

// 生成器创建无限序列
function* fibonacci() {
  let [a, b] = [0, 1];
  while (true) { yield a; [a, b] = [b, a + b]; }
}
```

## 异步迭代 (ES2018)

```js
async function* asyncGen() {
  for (let i = 0; i < 3; i++) {
    yield await fetchData(i);
  }
}
for await (let item of asyncGen()) {
  console.log(item);
}
```

`Symbol.asyncIterator` 是异步可迭代协议的 symbol。

## 容易弄错的点

1. **生成器是一次性的**：一旦耗尽（done: true），再次调用 `next()` 仍返回 `{ done: true }`，无法重置
   ```js
   function* g() { yield 1; }
   const gen = g();
   gen.next(); // { value: 1, done: false }
   gen.next(); // { value: undefined, done: true }
   gen.next(); // { value: undefined, done: true } -- 仍然是 done
   ```

2. **`for/of` 无法检测迭代器是否中途耗尽**：如果手动调用 `.next()` 让迭代器部分前进，`for/of` 从当前状态继续

3. **首次调用 `g.next(arg)` 传入的参数会被忽略**：第一个 `next()` 的参数没有对应的 `yield` 接收。从第二个 `next()` 开始，参数才被用作上一个 `yield` 的返回值

4. **`yield*` 的返回值**：`let result = yield* iterable` — 当被委托的生成器 `return` 一个值时，该值赋值给 `result`。但 `for/of` 循环不会捕获这个返回值

5. **普通对象不可迭代**：`for (let v of {x:1})` → TypeError。需要通过 `[Symbol.iterator]` 方法实现

6. **生成器函数返回的是生成器对象，不是直接值**：`const gen = genFn()` 不执行函数体，只返回生成器对象。调用 `gen.next()` 才开始执行

7. **异步迭代器的 `next()` 返回 `Promise<{value, done}>`**：`for await/of` 自动处理 Promise 封装，手动迭代则需 `await it.next()`
