# 第1章 JavaScript简介

JavaScript 是一门高级、动态、解释型编程语言，非常适合面向对象和函数式编程风格。

## 核心语言 vs 宿主环境

- **核心语言**：定义最小限度的 API（数值、文本、数组、集合、映射），不含输入/输出
- **宿主环境**：提供输入/输出及复杂特性（网络、存储、图形）。浏览器和 Node.js 是最常见的宿主

## 语言特性

- 基于原型的面向对象
- 函数是第一类对象，支持函数式编程
- 动态类型，运行时检查
- 单线程事件驱动模型

## 表达式与语句

- **表达式**：计算值，不改变程序状态
- **语句**：没有值，但改变程序状态
- 函数是一块可重复调用的代码块

## 在页面中加入 JS

1. 内嵌 `<script>...</script>`
2. 外链 `<script src="...">`

- `async`：异步下载，下载完立即执行，不保证顺序
- `defer`：异步下载，HTML 解析完后按顺序执行

3. 内联事件处理器（不推荐）
  第一，它将 HTML 与 JavaScript 强耦合，违反关注点分离原则，可维护性较差；第二，它依赖全局作用域，不利于 ES Module 和现代工程化；第三，它无法充分利用 DOM 事件模型，例如不能方便地支持事件委托、多个监听器、once、passive 等特性；第四，它存在安全风险，浏览器底层会将事件字符串动态编译执行，类似 new Function()，容易受到 XSS 攻击，同时很多生产环境启用 CSP 后会直接禁止内联脚本。因此现代开发普遍使用 addEventListener() 或框架提供的事件绑定方式。

   总结:

- **解耦**：HTML 与 JS 分离，符合关注点分离原则。
- **模块化**：不依赖全局作用域，支持 ES Module。
- **事件模型**：支持事件委托、多个监听器以及丰富的事件选项。
- **安全**：避免动态执行字符串，兼容 CSP，降低 XSS 风险。
- **可维护**：统一管理事件，便于调试、复用和后期维护。


| 对比项                           | `内联事件` | `addEventListener()` |
| ----------------------------- | ------ | -------------------- |
| HTML / JS 分离                  | ❌      | ✅                    |

| 可维护性                          | 差      | 好                    |
| **现代开发推荐**                    | ❌      | ✅                    |
