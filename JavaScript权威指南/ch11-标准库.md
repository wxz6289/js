# 第11章 JavaScript标准库

## Set 与 Map

### Set
```js
let s = new Set([1, 2, 3]);
s.add(4); s.delete(2); s.has(1);
s.size; s.clear();
for (let v of s) {}
s.forEach(v => {});
```

### Map
```js
let m = new Map([['key', 'value']]);
m.set('k', 'v'); m.get('k'); m.has('k');
m.delete('k'); m.size; m.clear();
for (let [k, v] of m) {}
```

### WeakSet / WeakMap
- 键必须是对象，弱引用（不阻止 GC）
- 不可迭代，无 size
- WeakMap 常用于存储私有数据或元数据

## 定型数组 (Typed Arrays)

- ES6，固定长度和类型：`Int8Array`、`Uint8Array`、`Uint8ClampedArray`、`Int16Array`、`Uint16Array`、`Int32Array`、`Uint32Array`、`Float32Array`、`Float64Array`、`BigInt64Array`、`BigUint64Array`
- 基于 ArrayBuffer

### ArrayBuffer
```js
let buf = new ArrayBuffer(16);           // 16 字节的缓冲区
let int32 = new Int32Array(buf);         // 作为 4个32位整数 视图
let int8 = new Int8Array(buf, 4, 8);    // 偏移 4 字节，取 8 个 8位整数
```

### 创建定型数组
```js
new Int32Array(10);                     // 10个元素
new Int32Array([1, 2, 3]);
Int32Array.of(1, 2, 3);
Int32Array.from(iterable);
new Int32Array(buffer, offset, length);  // 从 ArrayBuffer
```

### 方法
```js
arr.set(otherArr, offset);              // 批量设置
arr.subarray(start, end);               // 返回共享内存的视图
// 不改变长度的方法: sort, reverse, fill, copyWithin, map, filter...
// 无 push/pop/splice 等改变长度的方法
```

### 字节序
- 多数 CPU 是小端字节序
- 网络协议常为大端字节序
- DataView 可显式控制字节序：

```js
let dv = new DataView(buffer, offset, length);
dv.getInt32(offset, littleEndian);
dv.setUint32(offset, value, littleEndian);
```

## 正则表达式

```js

let re2 = new RegExp('pattern', 'flags');
```
- flags：`g`(全局)、`i`(忽略大小写)、`m`(多行)、`s`(dotAll)、`u`(Unicode)、`y`(粘滞)、`d`(索引)
- 方法：`test()`、`exec()`、String 的 `match/replace/search/split`

## 容易弄错的点

1. **Set 使用 `===` 判断重复，但 NaN 被视为等于 NaN**（与 `===` 行为不同！）：`new Set([NaN, NaN]).size` → `1`

2. **Map 的键比较也使用类似 `===` 的 SameValueZero 算法**：`NaN` 被视为等于 `NaN`，但 `+0` 和 `-0` 被视为相等（与 `Object.is` 不同）

3. **WeakMap 的键必须是对象**：`new WeakMap().set("str", 1)` → TypeError。且 WeakMap 不可迭代，没有 `size` 属性

4. **`Date.getMonth()` 从 0 开始**：1月=0，12月=11。这是极其常见的 bug 来源
   ```js
   new Date(2024, 0, 1)  // 2024年1月1日（0 = 1月！）
   new Date(2024, 11, 31) // 2024年12月31日
   ```

5. **`new Date(string)` 在不同浏览器中解析行为不同**：最好用 ISO 8601 格式 `"2024-01-01T00:00:00Z"` 或直接用数值参数

6. **JSON.stringify 遇到循环引用会抛 TypeError**：`JSON.stringify(window)` 报错。且 `undefined`、`function`、`Symbol` 会被静默忽略或转为 `null`

7. **正则的 `lastIndex` 属性**：带 `g` 或 `y` 标志的正则多次调用 `exec/test` 会从上次位置继续匹配，导致交替 `true/false`
   ```js
   const re = /foo/g;
   re.test('foo'); // true, lastIndex = 3
   re.test('foo'); // false! lastIndex 已到末尾
   ```

8. **定型数组溢出时不报错**：`new Uint8Array([256])` → `[0]`（截断取低8位）。`Uint8ClampedArray` 会箝位到 0-255 而非简单截断

9. **`setTimeout` 的最小延迟**：嵌套超过5层的 setTimeout 最小延迟为 4ms。零延迟不意味着立即执行
