# 第9章 类

## 类声明 (ES6)

```js
class ClassName {
  constructor(x) {
    this.x = x;              // 实例属性
  }
  method() {}                // 原型方法（不可枚举）
  static staticMethod() {}   // 静态方法
  get prop() {}              // getter
  set prop(val) {}           // setter
}
```
- Class 本质上是函数（`typeof ClassName === "function"`）
- 必须用 `new` 调用
- 类体自动为严格模式
- 类声明不提升（存在 TDZ）

## 实例字段与私有字段

```js
class MyClass {
  // 公有实例字段（ES2022）
  publicField = 1;

  // 私有字段（ES2022，前缀 #）
  #privateField = 2;

  #privateMethod() {
    return this.#privateField;
  }

  getPrivate() {
    return this.#privateMethod();
  }
}
```

## 静态字段

```js
class Config {
  static VERSION = '1.0';
  static #count = 0;         // 静态私有字段 (ES2022)
}
```

## 继承

```js
class Child extends Parent {
  constructor(x, y) {
    super(x);    // 必须先调用 super() 才能访问 this
    this.y = y;
  }

  method() {
    return super.method() + this.y;  // 调用父类方法
  }
}
```
- `[[HomeObject]]` 机制确保 `super` 指向正确的父类
- 静态方法同样可以通过 `super` 调用
- `extends` 可以继承内置类（Array、Error 等）
- `Symbol.species` 控制派生类创建的对象类型

## instanceof

```js
obj instanceof Class           // 检查原型链
Class[Symbol.hasInstance](obj) // 自定义 instanceof 行为
```

## 类型检测

```js
Object.prototype.toString.call(obj)  // 返回 "[object Type]"
// 可自定义 Symbol.toStringTag：
class X { get [Symbol.toStringTag]() { return 'X'; } }
```

## class vs prototype

| 特性 | class | prototype |
|------|-------|-----------|
| 语法 | 声明式 | 构造函数+prototype |
| 提升 | 否（TDZ） | 函数声明提升 |
| 严格模式 | 自动 | 需手动 |
| 方法可枚举 | 否 | 是（默认） |
| extends | 内置支持 | 手动实现原型链 |
| super | 内置 | 手动调用父类构造函数 |

## 容易弄错的点

1. **`super()` 必须在子类构造函数中使用 `this` 之前调用**，否则抛 ReferenceError。这是新手使用 extends 最常见的报错

2. **私有字段 `#field` 不能通过 `obj["#field"]` 或 `obj[varName]` 访问**：编译器直接拒绝。且子类不能访问父类的私有字段

3. **类方法是不可枚举的**：`Object.keys(instance)` 不会列出方法。这与传统 prototype 赋值不同

4. **`static` 方法中的 `this` 指向类本身，不是实例**：
   ```js
   class A {
     static create() { return new this(); } // this === A
   }
   ```

5. **`extends` 内置类（如 Array）**：在 ES6 前用 prototype 很难完美继承。class 语法可以，但要注意 `Symbol.species` 会影响 `map`/`filter` 返回的类型

6. **类字段初始化的执行顺序**：先 `super()` → 再按声明顺序执行实例字段初始化 → 最后执行 constructor 中的代码
   ```js
   class Child extends Parent {
     field = this.init(); // this 在这里已经可用 (super已调用)
   }
   ```

7. **`instanceof` 检查整个原型链**，不是只检查直接构造函数。跨 iframe/realm 的 `instanceof` 可能失效（不同执行上下文有不同的 Array/Object 构造函数）

8. **class 声明有 TDZ**：在声明前引用类名会抛 ReferenceError，不像函数声明有提升
