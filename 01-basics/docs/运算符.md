# 运算符与表达式

运算符作用于**运算元**（operand），对值进行运算并**返回一个值**。在 JavaScript 中，连赋值 `=`、逻辑与 `&&` 也是表达式。

词法结构、ASI、语句分类见 [grammar.md](grammar.md)；类型转换规则见 [types.md](types.md)。

---

## 1. 表达式

**表达式**会求值并产生一个值，可嵌套、可出现在任何「需要值」的位置。

### 1.1 分类

| 类型 | 示例 |
|------|------|
| 主表达式（Primary） | `42`、`"hi"`、`this`、`[1,2]`、`{a:1}`、`function(){}` |
| 成员访问 / 调用 | `obj.prop`、`arr[i]`、`fn(a, b)`、`new Foo()` |
| 运算符表达式 | `a + b`、`!x`、`a && b`、`a ?? b` |
| 条件（三元） | `cond ? a : b` |
| 逗号序列 | `(a, b, c)` — 结果为**最后一个**操作数的值 |

### 1.2 表达式 vs 语句

| | 表达式 | 语句 |
|---|--------|------|
| 目的 | 求值 | 执行动作 |
| 能否赋给变量 | 可以 | 一般不行 |
| 能否嵌套在另一表达式中 | 可以 | 不可以 |

```javascript
// 表达式语句：表达式 + 分号，求值后丢弃
(x + 1);
foo();

// 非法：语句不能出现在需要表达式的地方
// if (let x = 1) {}  ✗

// 合法：赋值是表达式
if ((x = 1)) {}
```

### 1.3 要点

- **所有运算符都返回值**，包括 `=`、`&&`、`||`、`??`、`,`。
- 赋值可链式：`a = b = 1`（结合性从右到左，见下文）。
- 函数体无显式 `return` 时返回 `undefined`。
- 一元运算符 `!`、`+`、`-`、`~`、`void` 或括号 `()` 可将函数声明转为函数表达式，从而支持 IIFE（见 `code/types/bool-iif.js`）。

## 2. 运算符分类

| 元数 | 说明 | 示例 |
|------|------|------|
| 一元 | 一个运算元 | `!x`、`++i`、`-n`、`typeof x` |
| 二元 | 两个运算元 | `a + b`、`a === b`、`a && b` |
| 三元 | 三个运算元 | `a ? b : c` |

同一符号在不同位置语义不同，例如 `-` 既是一元取负也是二元减法，`+` 既是一元转数字也是二元加/拼接。

---

## 3. 算术运算符

| 运算符 | 含义 |
|--------|------|
| `+` | 加 / 字符串拼接 |
| `-` | 减 |
| `*` | 乘 |
| `/` | 除 |
| `%` | 取余 |
| `**` | 幂（指数可为非整数） |

### `+` 的特殊规则

- **二元 `+`**：只要有一个运算元是字符串，另一个也会被转为字符串再拼接。二元 `+` 是唯一支持字符串的算术运算符。
- **一元 `+`**：对数字无效果；对非数字等价于 `Number(...)`。

其余算术运算符只对数字生效，运算元会被转为数字。

```javascript
1 + "2";     // "12"
+"-12";      // -12（转数字）
2 + null;    // 2
2 + undefined; // NaN
```

### 一元 `-` 与 `**`

一元 `-` 的优先级**高于** `**`，但语法上**不允许**一元运算符紧挨在 `**` 表达式左侧而不加括号：

```javascript
// -2 ** 2;     // SyntaxError
(-2) ** 2;     // 4
-(2 ** 2);     // -4
2 ** 3 ** 2;  // 512，即 2 ** (3 ** 2)，`**` 从右到左结合
```

## 4. 比较运算符

| 运算符 | 说明 |
|--------|------|
| `<` `<=` `>` `>=` | 关系比较（会类型转换） |
| `==` `!=` | 相等 / 不等（会类型转换） |
| `===` `!==` | 严格相等 / 严格不等（不转换类型） |
| `in` | 属性是否在对象或其原型链上 |
| `instanceof` | 对象是否为某构造函数的实例 |

### `==` 与 `===`

- 类型相同：比较值是否相等。
- 类型不同（仅 `==`）：按规则强制转换后再比较。
- 若任一值可能是 `true`、`false`、`0`、`''`、`[]` 等「易隐式转换」的值，**避免使用 `==`**，优先 `===`。

`null == undefined` 为 `true`；`NaN` 与任何值（含自身）用 `==` / `===` 均不相等，可用 `Object.is(NaN, NaN)` 或 `Number.isNaN(x)`。

## 5. 逻辑运算符

| 运算符 | 含义 | 短路 |
|--------|------|------|
| `&&` | 逻辑与 | 左假则不再求右 |
| `\|\|` | 逻辑或 | 左真则不再求右 |
| `??` | 空值合并 | 左为 `null`/`undefined` 才求右 |

### 返回值

逻辑运算符返回的是**运算元本身**，不一定是布尔值：

```javascript
0 && (x = 1);   // 0，x 不变
1 && (x = 1);   // 1，x 被赋值
null ?? "default"; // "default"
0 ?? "default";    // 0（仅 null/undefined 触发 ??）
```

### `??` 与 `||` / `&&` 不可混用

不加括号时，`??` 不能与 `||` 或 `&&` 出现在同一表达式中，否则 **SyntaxError**：

```javascript
// a || b ?? c;   // SyntaxError
(a || b) ?? c;   // 合法
a || (b ?? c);   // 合法
```

## 6. 位运算符

| 运算符 | 含义 |
|--------|------|
| `&` | 按位与 |
| `\|` | 按位或 |
| `^` | 按位异或 |
| `~` | 按位非 |
| `<<` `>>` `>>>` | 左移 / 有符号右移 / 无符号右移 |

位运算符先将运算元转为 32 位整数再运算。

**技巧**：双波浪号 `~~n` 可快速截断小数（等同 `Math.trunc` 对有限数的部分场景），不进行四舍五入：

```javascript
~~3.7;    // 3
~~(-3.7); // -3
```

## 7. 赋值运算符

| 形式 | 等价于 |
|------|--------|
| `=` | 直接赋值 |
| `+=` `-=` `*=` `/=` `%=` `**=` | 复合赋值 |
| `<<=` `>>=` `>>>=` `&=` `^=` `\|=` | 位运算复合赋值 |
| `&&=` `\|\|=` `??=` | 逻辑赋值（ES2021） |

赋值表达式**从右到左**结合，且返回赋值后的值：

```javascript
let a, b, c;
a = b = c = 1;  // 等价于 a = (b = (c = 1))
```

### 成员访问 vs 赋值优先级

`.` 与 `[]` 的优先级**高于**赋值，因此复合赋值会先算完左侧引用，再赋值：

```javascript
var o = { n: 1 };
var p = o;
o.x = o = { n: 2 };
// 1. 先求 o.x 的引用（此时 o、p 仍指向 { n: 1 }）
// 2. 再求 o = { n: 2 }，o 改指向新对象
// 3. 把新对象赋给步骤 1 保存的引用 → p.x = { n: 2 }

console.log(o.n, p.n); // 2, 1
console.log(o.x, p.x); // undefined, { n: 2 }
```

## 8. 其他运算符

| 运算符 | 说明 |
|--------|------|
| `? :` | 条件（三元），从右到左结合 |
| `,` | 逗号，从左到右，返回最后一项 |
| `++` `--` | 前置立即改值并返回新值；后置返回旧值再改值 |
| `typeof` | 返回类型字符串 |
| `void` | 求值后返回 `undefined` |
| `delete` | 删除对象属性，返回布尔 |
| `await` | 暂停 async 函数至 Promise  settled |
| `?.` | 可选链，左侧为 `null`/`undefined` 时短路为 `undefined` |
| `new` | 构造调用；无参时可写 `new Foo`，有参须 `new Foo()` |

### 三元运算符结合性

```javascript
false ? 1 : true ? 2 : 3;  // 2，即 false ? 1 : (true ? 2 : 3)
```

### 逗号运算符

常用于 `for` 循环头或需要「求值多个表达式但只保留最后一个」的场景：

```javascript
let x = (1, 2, 3);  // x === 3
```

## 9. 优先级与结合性

### 9.1 两个概念

| 概念 | 含义 | 类比 |
|------|------|------|
| **优先级**（precedence） | 不同运算符谁先结合 | 先乘除后加减 |
| **结合性**（associativity） | 优先级相同时从哪侧分组 | `a - b - c` → `(a - b) - c` |

优先级**数字越大越先算**（下表与 MDN / ECMA-262 一致）。

### 9.2 完整优先级表

|                | 优先级 | 结合性 | 运算符 |
|----------------|--------|--------|--------|
| 分组 | 20 | n/a | `( … )` |
| 成员 / 调用 / 构造 | 19 | 左 → 右 | `.` `[ … ]` `( … )` `` ` … ` `` `?.` |
| `new`（带参） | 19 | n/a | `new … ( … )` |
| `new`（无参） | 18 | 右 → 左 | `new …` |
| 后置更新 | 17 | n/a | `… ++` `… --` |
| 一元 | 16 | 右 → 左 | `!` `~` `+` `-` `++` `--` `typeof` `void` `delete` `await` |
| 幂 | 15 | 右 → 左 | `**` |
| 乘除模 | 14 | 左 → 右 | `*` `/` `%` |
| 加减 | 13 | 左 → 右 | `+` `-` |
| 移位 | 12 | 左 → 右 | `<<` `>>` `>>>` |
| 关系 | 11 | 左 → 右 | `<` `<=` `>` `>=` `in` `instanceof` |
| 相等 | 10 | 左 → 右 | `==` `!=` `===` `!==` |
| 按位与 | 9 | 左 → 右 | `&` |
| 按位异或 | 8 | 左 → 右 | `^` |
| 按位或 | 7 | 左 → 右 | `\|` |
| 逻辑与 | 6 | 左 → 右 | `&&` |
| 逻辑或 | 5 | 左 → 右 | `\|\|` |
| 空值合并 | 4 | 左 → 右 | `??` |
| 条件 | 3 | 右 → 左 | `? … : …` |
| 赋值 / 逻辑赋值 | 2 | 右 → 左 | `=` `+=` … `&&=` `\|\|=` `??=` |
| 逗号 | 1 | 左 → 右 | `,` |

> 不必背整张表。遇到复杂表达式，**加括号**或拆成多行；读他人代码时，从优先级最高、结合性最特殊的运算符入手。

### 9.3 易混淆场景

#### 一元 vs 二元

一元运算符优先级**高于**同符号的二元运算符：

```javascript
typeof !!x;     // typeof (!(!x))
- -"12";        // 12（两个一元负号）
// --"12";      // SyntaxError：-- 被解析为递减运算符
```

#### 成员访问与调用（左结合）

```javascript
a.b.c;          // (a.b).c
f()(g);         // (f())(g)
obj?.a?.b;      // 任一步为 null/undefined 则整体为 undefined
```

#### 逻辑运算符顺序

优先级：`&&` > `||` > `??`

```javascript
0 && 1 || 2;    // 2，即 (0 && 1) || 2
1 && 0 || 2;    // 2
1 || 0 && 2;    // 1，即 1 || (0 && 2)
```

#### 三元与赋值

三元优先级**低于**逻辑运算符，**高于**赋值：

```javascript
const ok = cond ? a : b;   // 无需括号
// a ? b : c = d;         // 解析为 (a ? b : c) = d，通常不是想要的
a ? b : (c = d);           // 明确意图时加括号
```

#### `new` 与成员访问

`new Foo()` 的优先级使得 `new Foo().bar` 解析为 `(new Foo()).bar`，而非 `new (Foo.bar())`。

#### 行首 `{` 与 `+`

语句上下文中，行首 `{` 可能被解析为块而非对象字面量，导致与 `+` 组合时出现反直觉结果（见 [types.md](types.md) 中 `[] + {}` / `{} + []`）。

#### 记忆口诀（粗略）

```
括号成员 new 调用，
一元更新再幂运算，
乘除加减移位比，
相等位运算逻辑串，
三元赋值逗号最末站。
```

## 10. 常用技巧

| 技巧 | 说明 |
|------|------|
| `x \|\| default` | `x` 为假值时用默认值（`0`、`''` 也会触发） |
| `x ?? default` | 仅 `null` / `undefined` 时用默认值 |
| `x && doSomething()` | 条件执行 |
| `cond ? a : b` | 替代简单 if/else 表达式 |
| `~~n` / `\|0` | 快速转 32 位整数（注意溢出） |
| `!!x` | 转布尔 |
| `(a, b, c)` | 顺序求值，取最后一项 |

## 11. 结构关系

```
表达式
 ├─ 主表达式（字面量、this、数组、对象、函数）
 ├─ 成员访问 / 调用 / new
 └─ 运算符表达式
      ├─ 一元 / 更新
      ├─ 算术 / 位 / 比较
      ├─ 逻辑（&& || ??）
      ├─ 三元 ? :
      ├─ 赋值
      └─ 逗号 ,
```

表达式加上分号（或在特定位置省略分号）即构成**表达式语句**；详见 [grammar.md](grammar.md)。
