# 词法、表达式与语句

JavaScript 程序由**词法单元**组成，经解析后形成**表达式**和**语句**两层结构。表达式求值，语句执行。

运算符细节见 [运算符.md](运算符.md)；标签、break、with 等见 [语句.md](语句.md)。

## 词法结构

### 字符与标识符

- 基于 **Unicode**（UTF-16），**大小写敏感**
- 标识符：首字符为字母、`$`、`_` 或 Unicode 字母；后续可加数字
- **保留字**不可作标识符（`let`、`class`、`return` 等）

### 字面量


| 类型     | 示例                       |
| ------ | ------------------------ |
| 数值     | `42`、`3.14`、`0xff`、`1e6` |
| 字符串    | `"hi"`、`'hi'`、`hi`       |
| 模板字符串  | `line\n${expr}`          |
| 正则     | `/pattern/flags`         |
| 布尔 / 空 | `true`、`false`、`null`    |


### 注释与空白

```js
// 单行注释
/* 多行注释，不可嵌套 */
```

空格、制表符、换行通常被忽略；换行在特定位置会触发 **ASI**（自动分号插入）。

### ASI（自动分号插入）

引擎在以下情况自动补 `;`：

1. 下一行 token 无法与当前行合法拼接
2. 输入结束
3. 部分受限产生式（如 `}` 前）

常见陷阱：`return`、`throw`、`break`、`continue` 后**直接换行**会插入分号：

```js
// 实际执行 return;  → undefined
return
  { x: 1 };

// 正确
return { x: 1 };
```

> 建议显式写分号，不依赖 ASI。

## 表达式

**表达式**会求值并产生一个值，可嵌套、可出现在需要值的位置。

### 分类


| 类型        | 示例                                                |
| --------- | ------------------------------------------------- |
| 主表达式      | `42`、`"hi"`、`this`、`[1,2]`、`{a:1}`、`function(){}` |
| 成员访问 / 调用 | `obj.prop`、`arr[i]`、`fn(a, b)`、`new Foo()`        |
| 运算符表达式    | `a + b`、`!x`、`a && b`、`a ?? b`                    |
| 条件（三元）    | `cond ? a : b`                                    |


### 要点

- JS 中**所有运算符都返回值**（包括 `=`、`&&`、`||`）
- 赋值表达式可链式：`a = b = 1`
- 逗号表达式 `a, b, c` 求值结果为**最后一个**操作数的值
- 函数体中，若未显式 `return`则返回undefined

## 语句

**语句**是程序的执行单元，以分号（或 `}`）结束，本身不构成可复用的值（表达式语句除外）。

### 常见语句


| 类别    | 示例                                               |
| ----- | ------------------------------------------------ |
| 表达式语句 | `x = 1;`、`foo();`、`++i;`                         |
| 块语句   | `{ let x = 1; ... }`                             |
| 声明    | `let x = 1;`、`function f() {}`、`class C {}`      |
| 条件    | `if`、`switch`                                    |
| 循环    | `for`、`while`、`do...while`、`for...of`、`for...in` |
| 跳转    | `break`、`continue`、`return`、`throw`              |
| 其他    | `try/catch`、`debugger`、`with`（已废弃）               |


### 表达式 vs 语句


|              | 表达式    | 语句   |
| ------------ | ------ | ---- |
| 目的           | 求值     | 执行动作 |
| 能否赋值给变量      | 可以（其值） | 一般不行 |
| 能否嵌套在另一个表达式中 | 可以     | 不可以  |


```js
// 表达式语句：表达式 + 分号
(x + 1);        // 求值后丢弃
foo();          // 调用是表达式，加分号变语句

// 以下非法：语句不能出现在需要表达式的地方
// if (let x = 1) {}  ✗
if ((x = 1)) {}  // ✓ 赋值表达式
```

### 声明语句的特殊性

`let` / `const` / `function` / `class` 声明是语句，但其中的初始化部分可以是表达式：

```js
const n = 1 + 2;           // 右侧是表达式
function f(a = 1 + 2) {}  // 默认参数是表达式
```

## 代码块与隐式返回值

代码块 `{ ... }` 本身不是表达式，但在**函数体**或 `eval` 字符串中，块内最后一条语句的值会成为隐式返回值：

```js
var a, b;
a = eval(`if (true) {
  b = 4 + 12;
}`);
// a === undefined（if 是语句，无返回值）
// b === 16
```

当前语法**不允许**把普通代码块的隐式返回值直接赋给变量；需要返回值应使用函数或显式 `return`。

## 结构关系

```
源码
 └─ 词法单元（token）
      └─ 解析
           ├─ 表达式  → 求值
           └─ 语句    → 执行
                └─ 块 { 语句... }
```
