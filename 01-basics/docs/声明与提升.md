# 声明与提升

变量、函数、类等标识符通过**声明**引入绑定；引擎在代码执行前会先处理这些声明，这一过程称为**提升**（hoisting）。理解「如何声明」与「提升后处于什么状态」，是读懂作用域、TDZ 和闭包行为的基础。

## 作用域概览

ES6 之前只有**全局作用域**和**函数作用域**；需要块级作用域时，通常用 IIFE 模拟。ES6 起原生支持**块级作用域**（`let` / `const`）。

| 声明方式 | 作用域 | 块级 `{}` |
| -------- | ------ | --------- |
| `var` | 函数 / 全局 | 忽略，不形成块级作用域 |
| `let` / `const` | 块级 | 尊重 `{}` 边界 |
| `function`（顶层） | 函数 / 全局 | 见下文「块级函数」 |
| `class` | 块级 | 尊重 `{}` 边界 |

## var / let / const 对比

| | `var` | `let` | `const` |
| --- | --- | --- | --- |
| 作用域 | 函数 / 全局 | 块级 | 块级 |
| 提升 | 有，初始化为 `undefined` | 有，处于 TDZ | 有，处于 TDZ |
| 声明前访问 | 返回 `undefined` | `ReferenceError` | `ReferenceError` |
| 重复声明（同作用域） | 允许，后者覆盖前者 | 不允许 | 不允许 |
| 重新赋值 | 允许 | 允许 | 不允许 |
| 声明时初始化 | 可选 | 可选 | **必须** |
| 全局声明 | 可能成为 `globalThis` 属性 | 仅词法绑定，可遮蔽但不覆盖 | 同 `let` |

### const 的绑定与对象

`const` 保证的是**绑定**不可改，引用类型的**属性**仍可修改：

```js
const obj = { a: 1 };
obj.a = 2;   // ✓ 修改属性
obj = {};    // ✗ 重新绑定
```

### 实践建议

- 默认用 `const`，需要重新赋值时再用 `let`
- 避免使用 `var`（除非维护旧代码）

## 声明提升

JavaScript 在代码执行前会先处理声明。不同声明方式的提升行为不同：

| 声明方式 | 是否提升 | 提升后状态 | 声明前访问 |
| -------- | -------- | ---------- | ---------- |
| `var` | ✅ | 初始化为 `undefined` | 返回 `undefined` |
| `let` | ✅ | 已创建绑定，未初始化（TDZ） | `ReferenceError` |
| `const` | ✅ | 已创建绑定，未初始化（TDZ） | `ReferenceError` |
| `function` | ✅ | 已完成初始化（含函数体） | 可以调用 |
| `class` | ✅ | 已创建绑定，未初始化（TDZ） | `ReferenceError` |

### var：声明与赋值分离

`var` 的提升把**声明**和**赋值**拆开处理：

1. **声明**（`var x`）提升到当前函数或全局作用域顶部
2. **赋值**留在原位置执行

```js
console.log(a); // undefined（已声明，未赋值）
var a = 1;
```

等价于：

```js
var a;
console.log(a); // undefined
a = 1;
```

- 作用域：函数级或全局级，**忽略** `{}` 块
- 函数体内外均可访问，初始化前值为 `undefined`

### let / const 与临时死区（TDZ）

`let` / `const` 的绑定**会提升**，但不会像 `var` 那样初始化为 `undefined`。从块（或作用域）开始到声明语句执行之前，绑定处于**临时死区**（Temporal Dead Zone），访问会抛出 `ReferenceError`——`typeof` 也不例外：

```js
console.log(b); // ReferenceError
let b = 1;

typeof undeclared; // "undefined"（真正未声明）
typeof c;          // ReferenceError（已提升但处于 TDZ）
let c;
```

### 函数声明提升

`function` 声明**整体提升**（含函数体），可在声明前调用：

```js
foo(); // ✓
function foo() {}
```

同作用域内重复声明时，**后者覆盖前者**。

> **函数表达式**（`const fn = function () {}`）遵循变量规则，不整体提升。

ES6 起，块内 `function` 声明具有块级作用域（具体行为因环境而异，严格模式下更一致）。ES6 之前，函数声明整体提升到作用域顶部。

### class 声明

`class` 与 `let` / `const` 类似：绑定会提升，但处于 TDZ，声明前不可访问。

## 重复声明

| 场景 | 是否允许 |
| ---- | -------- |
| 同作用域多次 `var` | ✓ 后者覆盖前者 |
| 同作用域多次 `function` | ✓ 后者覆盖前者 |
| `let` / `const` 与同名的 `var` / `let` / `const` | ✗ 语法错误 |

## 全局作用域

| 声明方式 | 是否成为 `globalThis` 属性 |
| -------- | -------------------------- |
| `var` | ✓ 可能覆盖已有全局属性 |
| `let` / `const` | ✗ 仅创建词法绑定，可遮蔽但不覆盖 |
| `function`（顶层） | ✓ 同 `var` |
| `class`（顶层） | ✗ 同 `let` |

## 未声明的变量

| 模式 | 读取未声明变量 | 赋值给未声明标识符 |
| ---- | -------------- | ------------------ |
| 严格模式 | `ReferenceError` | `ReferenceError` |
| 非严格模式 | `ReferenceError` | 隐式创建全局变量（可用 `delete` 删除） |

## 循环中的块级绑定

普通 `for` 循环的更新表达式会**重新赋值**同一变量，因此不能用 `const` 作循环变量。

`for...in` / `for...of` 每次迭代创建**新的块级绑定**，可使用 `const`，也避免了 `var` 在闭包中的经典陷阱：

```js
// var：所有回调共享同一个 i
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0); // 3, 3, 3
}

// let：每次迭代独立绑定
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0); // 0, 1, 2
}
```

## 相关语法：spread / rest（`...`）

同一运算符，语义由**位置**决定：

| 场景 | 模式 | 示例 |
| ---- | ---- | ---- |
| 函数调用 | 展开 | `fn(...args)` 等价于 `fn.apply(null, args)` |
| 数组 / 对象字面量 | 展开 | `[...arr]`、`{ ...obj }` |
| 函数声明 | 收集 | `function fn(...rest) {}` 替代 `arguments` |
| 解构赋值 | 收集 | `const [a, ...rest] = arr` |
