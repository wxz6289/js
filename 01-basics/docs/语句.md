# 语句

**语句**是程序的执行单元：它驱动控制流、声明绑定、处理异常。语句以分号或 `}` 结束，本身通常不能作为值嵌套进表达式（表达式语句是例外）。

表达式与语句的区别见 [运算符.md](运算符.md#1-表达式)；词法、ASI 见 [grammar.md](grammar.md)；变量声明与提升见 [声明与提升.md](声明与提升.md)。

---

## 1. 语句分类

| 类别 | 语句 | 说明 |
|------|------|------|
| 表达式语句 | `x = 1;`、`foo();`、`++i;` | 表达式 + 分号，求值后丢弃 |
| 块语句 | `{ ... }` | 组合多条语句，形成块级作用域 |
| 空语句 | `;` | 占位，本身无操作 |
| 声明 | `let` / `const` / `var`、`function`、`class` | 创建绑定或定义类型 |
| 条件 | `if`、`switch` | 分支 |
| 循环 | `while`、`do...while`、`for`、`for...in`、`for...of` | 迭代 |
| 跳转 | `break`、`continue`、`return`、`throw` | 改变控制流；可配合标签 |
| 标签 | `label: statement` | 为语句命名，供 break/continue 目标 |
| 异常 | `try` / `catch` / `finally` | 错误处理 |
| 其他 | `debugger`、`with`（已废弃） | 调试 / 历史语法 |

---

## 2. 表达式语句与块语句

### 2.1 表达式语句

任何合法表达式后加分号即构成语句：

```javascript
(x + 1);       // 求值后丢弃
foo();         // 函数调用
++i;           // 自增
a = b = 1;     // 赋值链
```

### 2.2 块语句

块 `{ ... }` 将多条语句组合在一起，并（在 `let`/`const`/`class` 下）形成**块级作用域**：

```javascript
{
  let x = 1;
  console.log(x);
}
// console.log(x); // ReferenceError
```

`if`、`for`、`while` 等后的 `{ }` 不是语法强制要求，但**强烈建议始终使用**，避免悬挂 `else` 等问题（见 §3.1）。

### 2.3 空语句

单独一个分号 `;` 是空语句，常见于 `for` 循环体占位：

```javascript
for (let i = 0; i < n; i++);  // 空循环体
```

## 3. 条件语句

### 3.1 if / else if / else

```javascript
if (condition) {
  // ...
} else if (other) {
  // ...
} else {
  // ...
}
```

`condition` 必须是表达式；`if (let x = 1)` 非法，但 `if ((x = 1))` 合法。

#### 悬挂 else（Dangling else）

无花括号时，`else` 总是与**最近的**未配对 `if` 绑定：

```javascript
if (a)
  if (b)
    console.log('a && b');
  else
    console.log('绑到内层 if');  // 不是外层 if 的 else
```

始终加 `{ }` 可消除歧义。示例见 `code/if-block.js`。

### 3.2 switch

```javascript
switch (expression) {
  case value1:
    // ...
    break;
  case value2:
    // ...
    break;
  default:
    // ...
}
```

| 要点 | 说明 |
|------|------|
| **全等匹配** | `case` 使用 `===` 比较 |
| **穿透（fall-through）** | 无 `break` 时继续执行下一 `case` |
| **`default` 位置** | 可在任意位置，仍按顺序匹配 |
| **多条 `case` 共用体** | 多个 `case` 标签可指向同一代码块 |

```javascript
switch (typeof x) {
  default:
    console.log('other');
    break;
  case 'string':
    console.log('string');
  case 'bigint':          // 无 break → 穿透
    console.log('string or bigint');
    break;
}
```

`switch` 的 `case` / `default` 子句共享同一外层块作用域；`let`/`const` 在不同子句中重复声明会报错，可用块 `{ }` 包裹各子句。

## 4. 循环语句

### 4.1 while 与 do...while

```javascript
while (condition) { /* ... */ }

do {
  /* 至少执行一次 */
} while (condition);
```

### 4.2 for

```javascript
for (initialization; condition; afterthought) {
  // ...
}
```

三部分均可省略；`for (;;)` 表示无限循环。

**初始化中的 `let`/`const`**：每次循环迭代使用同一绑定（与 `for...of` 中 `let` 每次迭代新建绑定不同，见下）。

### 4.3 for...in 与 for...of

| 形式 | 遍历 | 典型用途 |
|------|------|----------|
| `for...in` | 对象**可枚举属性名**（含原型链） | 对象键（不推荐用于数组） |
| `for...of` | **可迭代对象**的值 | 数组、Map、Set、字符串等 |

```javascript
for (const key in obj) { /* key 为字符串 */ }
for (const val of arr)  { /* val 为元素值 */ }
```

`for...in` 遍历数组会得到索引字符串，且可能包含继承属性，遍历数组优先 `for...of` 或数组方法。

### 4.4 循环中的 let 与 var

`for` 头部 `let i` 在**每次迭代**创建新的块级绑定（`for...in` / `for...of` 同理），闭包中可正确捕获当前 `i`：

```javascript
const funcs = [];
for (let i = 0; i < 5; i++) {
  funcs.push(() => console.log(i));
}
funcs[3](); // 3
```

`var` 只有**一个**函数级绑定，闭包共享最终值。详见 `code/let-for.js`、`code/fn-in-loop.js`。

## 5. 跳转语句

### 5.1 break

退出**最近**的 `switch` 或循环（`for` / `while` / `do...while` / `for...in` / `for...of`）。

```javascript
break;           // 退出当前循环 / switch
break outer;     // 退出标签 outer 所标记的语句
```

| 规则 | 说明 |
|------|------|
| 标签 | `break` 与标签之间**不能换行**（否则 ASI 插入分号，标签被忽略或报错） |
| 标签必须存在 | 否则 `SyntaxError` |
| 不能跳出函数 | `break` 只能作用于循环 / switch / 带标签语句，不能替代 `return` |

### 5.2 continue

跳过当前循环迭代剩余部分，进入下一次迭代。

```javascript
continue;
continue outer;  // 跳到标签 outer 所标记循环的下一次迭代
```

`continue` 同样不能跨函数边界；与标签之间不能换行。

### 5.3 return

结束**当前函数**执行并返回（可选）一个表达式值：

```javascript
return;
return expression;
```

#### ASI 陷阱

`return` 后**直接换行**会插入分号，导致返回值丢失：

```javascript
return
  { x: 1 };   // 实际 return; → undefined

return { x: 1 }; // 正确
```

`throw`、`break`、`continue` 同理。详见 [grammar.md](grammar.md#asi自动分号插入)。

### 5.4 throw

抛出异常，中断正常控制流，由最近的 `catch` 或传播至调用栈：

```javascript
throw new Error('message');
throw '任意值';  // 可以，但不推荐
```

## 6. 标签语句

语法：`label: statement`

为**任意语句**（通常是循环）命名，供 `break label` / `continue label` 使用。

```javascript
outer: for (let i = 0; i < 3; i++) {
  inner: for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) break outer;
  }
}
```

| 要点 | 说明 |
|------|------|
| 命名空间 | 标签与变量、函数**不在同一命名空间**，同名不冲突 |
| 作用域 | 标签有作用域，只在当前块 / 函数内可见 |
| 多标签 | 一条语句可有多个标签：`a: b: statement` |
| 限制 | **不能**用在 `let` / `const` 声明前（`label: let x = 1` 非法） |
| 使用场景 | 多层循环跳出；现代代码更推荐提取函数 + `return` |

示例见 `code/label.js`。

## 7. try / catch / finally

```javascript
try {
  // 可能抛错的代码
} catch (err) {
  // 处理 err
} finally {
  // 无论是否抛错都会执行
}
```

| 要点 | 说明 |
|------|------|
| `catch` 绑定 | `catch (e)` 中的 `e` 是块级绑定（ES2019+ 可省略 `(e)`） |
| `finally` | 常用来释放资源；若 `finally` 中有 `return`，会覆盖 `try`/`catch` 的返回值 |
| 可选部分 | `try/finally` 无 `catch` 合法；`try/catch` 无 `finally` 合法 |

```javascript
try {
  return 1;
} finally {
  return 2;  // 最终返回 2
}
```

## 8. 声明语句（概要）

声明语句创建绑定，详细规则见 [声明与提升.md](声明与提升.md)。

| 语句 | 作用域 | 提升 |
|------|--------|------|
| `var` | 函数 / 全局 | 提升并初始化为 `undefined` |
| `let` / `const` | 块级 | 提升但处于 TDZ，声明前不可访问 |
| `function` | 函数 / 块（严格模式下块级） | 函数声明整体提升 |
| `class` | 块级 | 提升但处于 TDZ |

声明语句的初始化部分可以是任意表达式：

```javascript
const n = 1 + 2;
function f(a = 1 + 2) {}
```

## 9. with（已废弃）

```javascript
with (obj) {
  // 块内可直接访问 obj 的属性，如同局部变量
}
```

| 要点 | 说明 |
|------|------|
| 严格模式 | **禁止**使用，`SyntaxError` |
| 性能 | 阻碍引擎优化，运行较慢 |
| 作用域 | 创建临时作用域；属性查找先查对象，再查作用域链 |
| 声明 | 块内 `var` 仍绑定外层；`let`/`const` 是独立绑定，**不会**在 `obj` 上创建属性 |
| 优先级 | 块内 `let`/`const`/`var` 声明的绑定优先于 `obj` 上的同名属性 |

```javascript
let obj = { a: 12, c: 21 };
with (obj) {
  var b = 10;   // b 是外层 var
  let c = 30;   // 块内 let，遮蔽 obj.c
  console.log(a, b, c); // 12, 10, 30
}
console.log(b);         // 10
console.log(obj.c);     // 21（let c 未写入 obj）
```

**不要使用 `with`**；用解构或临时变量替代。示例见 `code/with.js`。

## 11. 控制流总览

```
                    ┌─────────────┐
                    │  顺序执行    │
                    └──────┬──────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
      ┌─────────┐    ┌──────────┐    ┌──────────┐
      │ if/switch│    │  循环     │    │ try/catch │
      └────┬────┘    └─────┬────┘    └─────┬────┘
           │               │               │
           │         break / continue       │
           │               │               │
           └───────────────┼───────────────┘
                           ▼
                    return / throw
                    （退出函数 / 抛异常）
```

## 12. 常见陷阱速查

| 场景 | 问题 | 建议 |
|------|------|------|
| `return` / `throw` 后换行 | ASI 插入分号，返回值丢失 | 与值写在同一行，或加括号 |
| `break label` 换行 | 标签失效 | `break` 与标签同一行 |
| `if` 省略 `{ }` | 悬挂 else、逻辑错误 | 始终使用块 |
| `switch` 缺 `break` | 意外穿透 | 有意穿透加注释；否则写 `break` |
| `for...in` 遍历数组 | 索引为字符串、含继承属性 | 用 `for...of` |
| 循环 + `var` + 闭包 | 所有闭包共享同一变量 | 用 `let` 或 IIFE |
| `with` | 严格模式报错、难优化 | 用解构代替 |
